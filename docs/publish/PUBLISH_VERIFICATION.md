## Issue found and fixed during rehearsal

The first `-DecompPath` repeat exposed that extraction honored the external
checkout while downstream generators and the Makefile still resolved the
clone-local `decomp/BattleShip-main` path. `build.ps1` now creates a validated
temporary directory junction at that expected path, runs the remaining pipeline
inside `try/finally`, and removes only that exact junction during cleanup. The
default network-fetch path is unchanged.

One discarded repeat also encountered a CMake cache whose absolute source path
belonged to a previously moved scratch directory. That was an isolated stale
build-cache condition, not a source or output mismatch. The final clean clone
had no such cache and passed the complete pipeline above.
